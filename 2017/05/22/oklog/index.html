oklog

发表于 2017-05-22 |

injecter负责写优化(WAL)，让storage node负责read优化: CQRS
与RocketMQ类似:

  • injecter = commit log
  • storage node = consume log

不同在于：storage node是通过pull mode replication机制实现，可以与injecter位于不同机器
而RocketMQ的commit log与consume log是
在同一台broker上的

kafka是R/W紧耦合，无法独立scale
而oklog decouples read from write，可以独立scale
